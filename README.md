# DevOps Operations Portfolio

This repository documents my hands-on DevOps and SRE learning journey from an
operations-first perspective.

## Focus Areas
- Operational runbooks as code
- Automation using scripting and design-first approach
- Incident analysis and postmortems
- CI/CD pipeline design
- Reliability and operational consistency

## Repository Structure
- `runbooks/` – Operational runbooks written and maintained as code
- `scripts/` – Automation and helper scripts
- `postmortems/` – Practice and sanitized incident postmortems
- `automation-designs/` – Automation design documents (flows, risks, failure modes)
- `pipelines/` – CI/CD pipeline designs and examples

## Guiding Principles
- Reduce manual operational toil
- Improve consistency and clarity
- Design automation before implementation
- Respect access boundaries and security

## Notes
All content in this repository is non-production, non-confidential,
and created strictly for learning and skill development.
